# バックエンド向け課題

このプロジェクト（`trading_text`）を使った、バックエンド（Django）の基礎を学ぶための課題です。
できるところから順番にチャレンジしてみましょう！

## 課題1: 教科書の「詳細ページ」を作ってみよう
現在のアプリでは教科書の一覧・検索はできますが、それぞれの教科書の詳細を見るページがありません。詳細ページを追加してみましょう。

### やること
1. **URLの追加**
   `trading_text/urls.py` または `main/urls.py`（設定されている方）を開き、詳細ページ用のURLパターンを追加します。
   例: `path('book/<int:book_id>/', views.book_detail, name='book_detail')`
2. **ビュー（View）の作成**
   `main/views.py` に新しく `book_detail(request, book_id)` という関数を作ります。
   データベースから指定された `book_id` に一致する `Book` データを取得し、テンプレートに渡して表示できるようにします。
   *ヒント: `Book.objects.get(id=book_id)` や `get_object_or_404(Book, id=book_id)` を使います。*
3. **テンプレート（HTML）の準備**
   （フロントエンドの作業になりますが）表示用のHTMLファイルを用意して、取得した本のタイトルや価格を表示してみましょう。

## 課題2: データベースのテーブル（モデル）に新しい項目を追加しよう
教科書を出品する際に、「その教科書の説明やアピールポイント」を書けるようにしたいです。

### やること
1. **モデルの変更**
   `main/models.py` を開き、`Book` クラスの中に `description` （説明文）というフィールドを追加してください。
   *ヒント: 長い文章を保存したい場合は `models.TextField()` を使います。*
2. **マイグレーションの実行**
   ターミナルで以下のコマンドを実行し、変更したモデルをデータベースに反映させます。
   ```bash
   python manage.py makemigrations
   python manage.py migrate
   ```
3. **管理画面での確認**
   （必要であれば）`main/admin.py` に `Book` モデルを登録して、Djangoの管理画面（`/admin`）から説明文が入力できるようになったか確認してみましょう。

---

## 課題3: 出品機能（データの保存）を完成させよう
現在、`main/views.py` の `listing_form` 関数はただ画面を表示するだけになっています。フォームから送られたデータを受け取ってデータベースに保存する処理を書いてみましょう。

### やること
1. **POSTリクエストの処理**
   `listing_form` 関数の中で、`request.method == 'POST'` だった場合の処理を追加します。
2. **データの受け取りと保存**
   リクエストから `title`, `author`, `price` などのデータを受け取り、新しい `Book` オブジェクトを作成してデータベースに保存（`.save()`）します。
3. **リダイレクト**
   無事に出品（保存）が完了したら、トップページなど他の画面に遷移（リダイレクト）するようにしましょう。
   *ヒント: `django.shortcuts` の `redirect` を使います。*

---
わからないことがあれば、Djangoの公式ドキュメントを調べたり、既存のコード（`get_search_context`など）を参考にしてみてください！
