# 提出手順：コミットとPull Request（PR）の作成

もしコードが書けて、手元での動作確認まで完了したら、以下の手順でGitHubに提出（Pull Requestを作成）しましょう！

## 1. コミットとプッシュ（ターミナルでの作業）
ターミナルで以下のコマンドを実行し、書いたコードをGitHubへアップロード（プッシュ）します。

```bash
# 変更したファイルをすべてコミットの準備状態（ステージング）にする
git add .

# 変更内容に名前（コミットメッセージ）をつけて保存する
# ※ "Bookモデルに説明文を追加" の部分は、自分の作業内容に合わせて変えてください
git commit -m "Bookモデルに説明文を追加"

# GitHub（リモートリポジトリ）に現在のブランチをプッシュする
# ※ feature/add-book-description の部分は、自分が作業しているブランチ名に合わせてください
git push origin feature/add-book-description
```

## 2. Pull Request（PR）の作成（GitHub上での作業）
プッシュが終わったら、ブラウザでGitHubのこのプロジェクトのページを開きます。

1. **PR作成画面を開く**
   画面の上部の方に黄色や緑色の帯で **「Compare & pull request」** というボタンが表示されているはずなので、それをクリックします。
   *(※ 表示されていない場合は、画面上の「Pull requests」タブを開き、緑色の「New pull request」ボタンを押して、`compare:` の部分を自分がプッシュしたブランチに変更してください)*

2. **内容を記入する**
   - **タイトル**: どんな機能を作ったか分かるように書きます。（例：「課題2: Bookモデルに説明文を追加」）
   - **コメント欄（Description）**: どんな変更をしたか、工夫した点や、レビューしてほしい点などを記入します。（例：「課題2完了しました。動作確認もOKです」など）

3. **提出する**
   最後に、右下にある緑色の **「Create pull request」** ボタンをクリックすれば提出完了です！
